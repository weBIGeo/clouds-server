Preprocessing server for real-time volumetric cloud rendering in the browser. Originally developed by Wendelin Muth as part of his bachelor thesis at TU Wien.

Real-Time Volumetric Rendering of Meteorological Cloud Data
Wendelin Muth, Research Unit of Computer Graphics, TU Wien, March 2026
Supervisor: Manuela Waldner

Built for the weBIGeo web-based geographic visualization project. It fetches weather forecast data from DWD (ICON-D2 model), processes it into compressed tile hierarchies, and serves them for WebGPU-based ray-marching cloud rendering.

The original codebase can be found here. This server runs a production-ready variant with customizable tile creation and purging strategies developed by Gerald Kimmersdorfer. You can see weBIGeo in action at webigeo.alpinemaps.org.

GET /status

Returns the current server status: what is actively being processed right now and when the next scheduled maintenance run is.

Response

{
  "version": "1.4",
  "status": "working",
  "next_maintenance": "2026-04-12T00:30:00Z",
  "active": {
    "2026041312_001": { "stage": "download", "detail": "133/522 files", "percent": 25 }
  },
  "tilesets": { "size": 4831838208, "max": 10737418240 }
}
FieldDescription
versionServer version string
status"idle" or "working" (based on actively running tasks)
next_maintenanceISO 8601 UTC datetime of the next scheduled purge + auto-build run
activeObject keyed by folder name (YYYYMMDDHH_SSS) with live stage, detail, and percent (0–100)
tilesets.sizeTotal bytes used by ready tile sets
tilesets.maxConfigured cache size limit in bytes
GET /tilesets

Returns all known tile sets from the database, optionally filtered by status. Sorted by target time ascending.

Query Parameters

ParameterDefaultDescription
statusallFilter by status: ready, pending, fetching, or failed

Response

{
  "items": [
    {
      "id": "2026040812",
      "folder": "2026040809_003",
      "status": "ready",
      "path": "/2026040809_003/",
      "run": "2026040809",
      "step": 3,
      "size": 123456789,
      "queued_at": "2026-04-08T09:00:00Z",
      "completed_at": "2026-04-08T09:15:42Z",

    }
  ]
}
FieldDescription
idTarget hour (YYYYMMDDHH)
folderFolder name on disk (YYYYMMDDHH_SSS)
statusready | pending | fetching | failed
pathURL prefix for tile and shadow files (only set when status=ready)
runModel run used (YYYYMMDDHH)
stepForecast step in hours from run to target
sizeTotal size in bytes (only set when status=ready)
queued_atISO 8601 UTC timestamp when first queued
completed_atISO 8601 UTC timestamp when marked ready (only set when status=ready)

GET /{folder}/tiles/{z}/{x}/{y}.ktx2

Serves a cloud tile in KTX2 format. {folder} is the value of the path field returned by /tilesets (e.g. 2026040809_003). The tiles sub-path is fixed.

GET /{folder}/shadow.ktx2

Serves the shadow map for the given tile set.

All other paths return 403 Forbidden.
